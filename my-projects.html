Project 1: University Project - Itinerary Manager

This project was made for my first larger University assignment.

The goal was to create a Java application using the Swing framework, for a hypothetical activity management company.

This was my first real experience with Java and Swing, before this I had only used Java for smaller assignments.

The criteria for the project had four phases, each with its own requirements. Each phase added more functionality to the application.

Phases one through three were developed as purely console based interactions, this takes the user through a series of inputs and options to manage their itinerary. At the end it then saves this itinerary to a file using a specific format, this file can then be loaded back into the application. The fourth phase was to create a GUI using the Swing framework, this was a challenge for me as I had never used Swing outside of practice, small tasks before. The GUI only had to be a very simple interface, it just had to show the itineraries, and allow the user to double click an itinerary for further information.

The project was a great learning experience, and I learned a lot about Java and Swing. Since it is a University project, I cannot share the code publicly, but I can share the results from it.

For this project, I recieved 70%, which when combined with the first assessment for the module, gave me a final mark of 75% for the module, which is a 1st class grade.

Project 2: University Project - SQL Query Development

GitHub Repository

This project was made for my second University assignment, which was focused on SQL and database management.

This project was my first time ever using SQL, and writing queries to a database.

The goal was to create a 'portfolio' of SQL queries related to a specified database, to improve our knowledge on how SQL and it's specific features work.

For this project, I used SQL Server, alongside the SQL Server Management Studio (SSMS) to write and test my queries. Outside of the project, I also practiced using the online platform LeetCode, which has a section dedicated to SQL queries, for this I decided to use MySQL, to broaden my knowledge some more.

For this project, I wrote over 40 queries, covering a wide variety of topics, such as joins, subqueries, aggregations, and functions.

The project was a great learning experience, and I learned a lot about SQL and how to write queries. Since it is a University project. The queries can be found in the GitHub repository.

Project 3: Personal Project - JavaScript To-Do App with Weather

GitHub Repository
Link to page

This project was my first attempt at making some kind of "web app" using JS. I wanted a simple app that would still stretch my knowledge somewhat, so I decided on creating a basic project, a to-do application.

This project would teach me how to dynamically create DOM elements with JavaScript, which is something I really I didn't know when I was creating my Cookie Clicker Clone.

I used my previous knowledge in using LocalStorage to save the tasks created and had to understand further how this JSON actually worked to be able to write my own loading function, it took some effort to work out, but I am pleased with how it came out.

The application allows the user to add tasks, tick them off, and remove them, as well as completely clear the tasks at the bottom.

As a challenge, once I'd finished the main functionality, I wanted to try and add some form of API usage, therefore I also added a simple weather indicator to the top, this takes the users location, and uses the langitude and longitude to ping the OpenWeather API for the users current weather.

This can't be seen on the site as it requires an API key, however screenshots below and a showcase in the repo show this working, as well as just adding an API key to the source code, would allow you to see for yourself.

The GitHub repo, as well as the hosted site can both be found at the top.
